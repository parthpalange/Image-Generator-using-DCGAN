## Overview

This project implements a **DCGAN (Deep Convolutional Generative Adversarial Network)** pipeline based on the [DCGAN paper](https://arxiv.org/abs/1511.06434) by Radford et al., extended to support **256×256 resolution** images. It provides a complete end-to-end workflow: from raw image preprocessing to model training, quantitative evaluation (FID & IS), and inference with latent space exploration.

---

## Features

- 🖼️ **256×256 Resolution** — Extended DCGAN architecture with 7-layer Generator and Discriminator
- 📁 **Custom Dataset Support** — Bring your own images; supports JPEG, PNG, BMP, TIFF, WebP
- ⚙️ **Config-Driven** — All hyperparameters in a single YAML file
- 📊 **TensorBoard Logging** — Real-time loss curves and generated image grids
- 💾 **Checkpoint & Resume** — Save checkpoints at intervals; resume training from any checkpoint
- 📈 **Evaluation Metrics** — FID (Fréchet Inception Distance) and IS (Inception Score)
- 🎨 **Latent Space Interpolation** — Spherical interpolation (slerp) between latent vectors
- 🔧 **Spectral Normalization** — Optional SN on the Discriminator for training stability
- 🏷️ **Label Smoothing** — One-sided label smoothing for more stable GAN training
- 🛡️ **Robust Error Handling** — Gracefully skips corrupt images; never crashes mid-training

## Installation

### Prerequisites

- Python 3.10+

## Usage

### 1. Preprocessing

Prepares your raw images for training by resizing and converting them.

```bash
python src/preprocess.py --config config/default.yaml
```

**What it does:**
- Recursively scans `data/raw/` for images (JPEG, PNG, BMP, TIFF, WebP)
- Resizes to 256×256 using Lanczos resampling
- Converts to RGB color space
- Saves processed images as PNGs to `data/processed/`
- Skips corrupt/unreadable images with a warning (won't crash)

**Output:**
```
Preprocessing images...
100%|██████████████████████████████| 5000/5000 [00:45<00:00, 110.23img/s]

Summary:
  Total found:     5000
  Processed:       4987
  Skipped:         13
  Time elapsed:    45.3s
```

### 2. Training

Train the DCGAN on your processed dataset.

```bash
# Start training from scratch
python src/train.py --config config/default.yaml

# Resume from a checkpoint
python src/train.py --config config/default.yaml --resume models/checkpoint_epoch_50.pth
```

**What it does:**
- Trains Generator and Discriminator adversarially using BCELoss
- Logs losses and sample images to TensorBoard
- Saves checkpoints every `save_interval` epochs (default: 10)
- Generates sample grids every `sample_interval` steps (default: 500)
- Uses a fixed noise vector for consistent progress visualization

**Monitor training with TensorBoard:**
```bash
tensorboard --logdir outputs/logs/
```

Then open http://localhost:6006 in your browser.

**Training output:**

### 3. Evaluation

Quantitatively evaluate your trained generator using standard GAN metrics.

```bash
python src/evaluate.py --config config/default.yaml \
    --checkpoint models/generator_final.pth \
    --num_images 1000
```

**Computed metrics:**
| Metric | Description | Good Score |
|--------|-------------|------------|
| **FID** (Fréchet Inception Distance) | Distribution similarity to real data | Lower is better (< 50 is decent) |
| **IS** (Inception Score) | Quality and diversity of generated images | Higher is better (> 2.0 is decent) |

**Output:**
```

║        Quality Report               ║
╠══════════════════════════════════════╣

║  Inception Score:     3.45 ± 0.21   ║
║  Images Generated:   1000           ║

```

Results saved to `outputs/evaluation/`.

### 4. Inference

Generate images from a trained model.

```bash
# Generate 16 random images
python src/inference.py --checkpoint models/generator_final.pth --num_images 16

# Reproducible generation with a seed
python src/inference.py --checkpoint models/generator_final.pth --num_images 16 --seed 42

# Latent space interpolation (smooth transitions between two random images)
python src/inference.py --checkpoint models/generator_final.pth --interpolate --steps 10

# Custom output directory
python src/inference.py --checkpoint models/generator_final.pth --num_images 64 --output outputs/my_samples/
```

Generated images are saved to `outputs/generated/` (or custom `--output` path).

## Architecture

### Generator (7 layers)

Maps a 100-dimensional noise vector to a 256×256 RGB image.

```
Input: z ∈ ℝ¹⁰⁰ ~ N(0, 1)
  │
  ├─ ConvTranspose2d → 1024 × 4 × 4    (Project & Reshape)
  ├─ ConvTranspose2d →  512 × 8 × 8     + BatchNorm + ReLU
  ├─ ConvTranspose2d →  256 × 16 × 16   + BatchNorm + ReLU
  ├─ ConvTranspose2d →  128 × 32 × 32   + BatchNorm + ReLU
  ├─ ConvTranspose2d →   64 × 64 × 64   + BatchNorm + ReLU
  ├─ ConvTranspose2d →   32 × 128 × 128  + BatchNorm + ReLU
  └─ ConvTranspose2d →    3 × 256 × 256  + Tanh
  │
Output: RGB image ∈ [-1, 1]
```

### Discriminator (7 layers)

Classifies 256×256 RGB images as real or fake.

```
Input: RGB image (3 × 256 × 256)
  │
  ├─ Conv2d →   64 × 128 × 128   + LeakyReLU(0.2)
  ├─ Conv2d →  128 × 64 × 64     + BatchNorm + LeakyReLU(0.2)
  ├─ Conv2d →  256 × 32 × 32     + BatchNorm + LeakyReLU(0.2)
  ├─ Conv2d →  512 × 16 × 16     + BatchNorm + LeakyReLU(0.2)
  ├─ Conv2d → 1024 × 8 × 8       + BatchNorm + LeakyReLU(0.2)
  ├─ Conv2d → 1024 × 4 × 4       + BatchNorm + LeakyReLU(0.2)
  └─ Conv2d →    1 × 1 × 1       + Sigmoid
  │
Output: P(real) ∈ [0, 1]
```

**Weight Initialization:** All conv weights initialized from N(0, 0.02) per the DCGAN paper.

## Configuration

All hyperparameters are in [`config/default.yaml`](config/default.yaml):

| Section | Parameter | Default | Description |
|---------|-----------|---------|-------------|
| `data` | `image_size` | 256 | Output image resolution |
| `data` | `channels` | 3 | RGB (3) or Grayscale (1) |
| `data` | `num_workers` | 4 | DataLoader worker processes |
| `model` | `latent_dim` | 100 | Noise vector dimensionality |
| `model` | `g_features` | 64 | Generator base feature maps |
| `model` | `d_features` | 64 | Discriminator base feature maps |
| `training` | `epochs` | 200 | Total training epochs |
| `training` | `batch_size` | 16 | Batch size (reduced for 256×256) |
| `training` | `lr_g` | 0.0002 | Generator learning rate |
| `training` | `lr_d` | 0.0002 | Discriminator learning rate |
| `training` | `beta1` | 0.5 | Adam β₁ (per DCGAN paper) |
| `training` | `beta2` | 0.999 | Adam β₂ |
| `training` | `label_smoothing` | 0.9 | Real label value (< 1.0 for stability) |
| `training` | `save_interval` | 10 | Checkpoint frequency (epochs) |
| `training` | `sample_interval` | 500 | Sample grid frequency (steps) |
| `training` | `device` | auto | `auto`, `cuda`, or `cpu` |

To customize, either edit `default.yaml` or create a new config:
```bash
python src/train.py --config config/my_custom_config.yaml
```

## Training Tips

### Dataset Size
| Dataset Size | Expected Quality | Recommendation |
|-------------|-----------------|----------------|
| < 500 | Poor | Consider data augmentation or DiffAugment |
| 500 – 2,000 | Moderate | Use label smoothing, may need longer training |
| 2,000 – 10,000 | Good | Default settings should work well |
| > 10,000 | Excellent | Can increase batch_size if VRAM allows |

### GPU Memory (VRAM) Requirements
| Batch Size | Approx. VRAM (256×256) |
|-----------|----------------------|
| 8 | ~4 GB |
| 16 | ~8 GB |
| 32 | ~14 GB |
| 64 | ~24 GB |

### Common Issues & Fixes

| Symptom | Cause | Fix |
|---------|-------|-----|
| Generator produces noise | Not enough training | Train for more epochs |
| Mode collapse (all images look same) | D too strong | Lower `lr_d`, increase `label_smoothing` |
| D loss → 0, G loss → ∞ | D overpowering G | Add noise to D inputs, use spectral norm |
| Training oscillates wildly | Learning rate too high | Lower both `lr_g` and `lr_d` |
| CUDA out of memory | Batch too large | Reduce `batch_size` in config |

### Enabling Spectral Normalization

For more stable training (especially on small datasets), enable spectral normalization on the Discriminator by modifying `train.py` to pass `use_spectral_norm=True`:

```python
disc = Discriminator(num_channels=channels, d_features=d_features, use_spectral_norm=True)
```

## Troubleshooting

### UnicodeDecodeError on Windows
If you encounter encoding errors when loading the config:
```
UnicodeDecodeError: 'charmap' codec can't decode byte...
```
This is already fixed in the codebase. All config files are opened with `encoding='utf-8'`.

### No CUDA Available
Ensure you have:
1. An NVIDIA GPU
2. [CUDA Toolkit](https://developer.nvidia.com/cuda-toolkit) installed
3. PyTorch installed with CUDA support:
```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

### Images Not Found During Preprocessing
Ensure your images are placed inside `data/raw/`. Subdirectories are supported:
```
data/raw/
├── category_a/
│   ├── img001.jpg
│   └── img002.png
└── category_b/
    └── img003.jpg
```
